What each code block covers
1. Linear Regression Basics — Fits y = 2x + 3 using nn.Linear(1,1), MSELoss, and SGD optimizer. This is the simplest possible PyTorch training loop: forward pass → compute loss → backward() (backprop) → optimizer.step() (update weights). Output shows loss dropping from ~0.014 to ~0.0115, and the learned weight (1.94) and bias (3.02) converging close to the true 2 and 3.
2. Dropout Network on MNIST (digit images) — A neural net with a hidden layer (784→128→10, since MNIST images are 28×28=784 pixels, 10 possible digits). nn.Dropout(p=0.5) randomly "turns off" 50% of neurons during training — this is a regularization technique that prevents the network from memorizing training data (overfitting) by forcing it to not rely too heavily on any single neuron.
3. Binary Classifier (Sigmoid output) — A single-layer network with Sigmoid() output for 2-class problems, trained with BCELoss (Binary Cross-Entropy — the standard loss for binary classification, same family as what you used with Logistic Regression on Day 3). Data here is synthetic: label = 1 if x1+x2 > 0. Loss steadily drops from 0.76 to 0.42 over 50 epochs.
4. Adam vs SGD Optimizer Comparison — Two identical models (same starting weights) trained one step each — one with Adam optimizer, one with SGD + momentum. This demonstrates that different optimizers exist for the same "update weights to reduce error" job — Adam adapts its learning rate automatically per-parameter, while SGD+momentum uses a simpler, more manual approach. Both start at the same loss (0.6895) since it's only the very first step.
5. Custom Layer (ElementwiseScaleLayer) — Shows how to build your own custom PyTorch layer by subclassing nn.Module. This layer just multiplies each input by a learnable weight — demonstrating that layers aren't limited to PyTorch's built-ins; you can define any custom operation with nn.Parameter (a tensor PyTorch will automatically learn/update).
6. Learning Rate Scheduler — StepLR automatically decreases the learning rate over time (0.1 → 0.01 → 0.001, every 5 epochs). Why: early in training you want big weight updates (fast learning); later, smaller updates help the model fine-tune without overshooting the best solution.
